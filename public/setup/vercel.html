No-terminal Vercel setup

This page is the browser-first self-host path for Vault MCP. It walks a non-terminal user through creating a Vercel-hosted MCP server, checking that it is alive, and copying the values back into the Obsidian plugin.

Current state

This is a guided private-alpha checklist. The final one-click template still needs a public/template repository and automated token handoff. Until then, account approvals in Vercel, Neon, GitHub, ChatGPT, or Claude may still appear in the browser.

What you are creating

A public HTTPS server that stores only the approved derived index from your vault and exposes MCP tools to AI clients.

What stays local

Your Obsidian vault stays on your computer. The plugin decides what can sync and applies approved write proposals locally.

What clients use

ChatGPT, Claude, Codex, and MCP Inspector connect to the MCP endpoint through OAuth or a minted access token, not the plugin sync token.

1

Open the Obsidian plugin first

In Obsidian, open Vault MCP and keep the `Start here` guide visible. That plugin page is the control center for this setup.

Settings -> Community plugins -> Vault MCP -> Start here
2

Create or choose a Vercel project

For private alpha, import the Vault MCP platform repository into Vercel. The future public version should replace this with a one-click deploy button that clones a template repository.

Open Vercel new project Read self-host docs
3

Add required environment values

Vercel needs a database, public URL, sync token, and OAuth values. Use strong generated values for secrets and keep them out of screenshots.

PUBLIC_BASE_URL, DATABASE_URL, MCP_SYNC_TOKEN, OAUTH_JWT_SECRET, OAUTH_AUTH_PASSWORD, OAUTH_SCOPES, ALLOWED_ORIGINS
4

Check the deployed server

After deployment, open the health URL in your browser. You want ok: true, storage ready, and a service MCP resource URL ending in /mcp.

https://your-vercel-domain.vercel.app/healthz
5

Copy these values back into the Obsidian plugin

The plugin needs the base server URL and admin sync token. The base URL does not include /mcp, /admin, or /oauth.

Server URL: https://your-vercel-domain.vercel.app
Sync token: paste your MCP_SYNC_TOKEN only into the plugin
6

Run plugin preflight, preview, and sync

Back in Obsidian, click `Check connection`, then `Preview index`. Review what is allowed, denied, or held for review before syncing.

Check connection -> Preview index -> Review queue -> Sync now
7

Connect ChatGPT or another MCP client

Clients use the MCP endpoint and OAuth. The endpoint includes /mcp. The sync token is only for the Obsidian plugin.

https://your-vercel-domain.vercel.app/mcp
Search my vault for active project notes and show one result card.
!

Recovery checklist

If something feels wrong, stop syncing first. Then rotate exposed tokens, revoke OAuth clients, delete the derived server index, or restore local notes from the plugin write-audit backup folder.

  • Disable the plugin in Obsidian Community plugins.
  • Rotate MCP_SYNC_TOKEN if it was exposed.
  • Rotate OAuth secrets or revoke clients if client access should reset.
  • Delete the server-side vault index if you no longer want remote search.
  • Restore from 00 System/Vault MCP Write Audit after approved local writes.
