Built like a firewall between your vault and the model.

The server never gives an AI client broad filesystem access. It exposes a filtered, rebuildable index made from notes that pass the source policy.

01 / SOURCE POLICY

Only approved notes enter the index.

Project homes, the Task Hub, and selected technical references are eligible. Sensitive areas, raw daily notes, finance, identity, credentials, and archived material stay out.

02 / PRIVATE CITATIONS

Results point back to controlled chunks.

Search returns concise matches with private citation URLs, metadata, headings, and Obsidian links so the answer can stay grounded.

03 / CLIENT READY

One remote MCP surface for many clients.

ChatGPT, Claude, Codex, and other MCP clients can talk to the same hosted endpoint through OAuth and Streamable HTTP.

From Obsidian graph to hosted MCP server.

The system is intentionally simple: scan the vault copy, filter aggressively, sync approved chunks, then answer MCP tool calls.

LOCAL

Indexer scans the vault snapshot.

Markdown files are parsed locally, split into useful chunks, and checked against the allowlist and denylist.

SYNC

Approved context is uploaded.

The remote database receives a full replacement index. Deleted or denied notes disappear on the next sync.

MCP

Clients call search and fetch.

The server validates auth, handles MCP JSON-RPC, and returns only allowlisted chunks.

The endpoint is the product.

The landing page is public. The useful data path is private. MCP clients connect to the hosted endpoint, complete OAuth, and receive exactly two tools: search for discovery and fetch for reading a selected chunk.

MCP endpoint https://vault-mcp-connector.vercel.app/mcp
  • POST handles MCP JSON-RPC requests.
  • GET supports authenticated SSE.
  • /wiki/ explains every major file and flow.

Designed to keep the vault as the source of truth.

The remote server is not a second vault. It is a controlled read surface backed by a derived index. If the policy changes or a note is removed, the index can be rebuilt from the local source.

Hard boundaries
  • No note editing through MCP.
  • No guessed filesystem paths.
  • No raw daily notes or credentials.
  • No finance, legal, identity, or review-gated material.
  • No broad vault browsing.
